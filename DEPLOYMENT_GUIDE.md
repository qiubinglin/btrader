# BTrader 订单流交易UI 部署指南

## 🎯 项目概述

本项目为您的C++交易系统提供了一个现代化的订单流交易界面，具备以下核心功能：

### 📊 订单流分析
- **实时K线图表**: 专业级价格走势展示
- **逐笔成交明细**: 市场微观结构分析  
- **买卖档位展示**: 深度订单簿可视化
- **足迹图分析**: 价格档位成交量分析
- **微盘口界面**: 最优买卖价快速交易

### 💼 交易管理
- **智能下单面板**: 多种订单类型支持
- **实时持仓监控**: 盈亏状态实时更新
- **订单生命周期管理**: 从下单到成交全程跟踪

## 🏗️ 系统架构

## 🚀 快速开始

### 环境要求
- **Node.js**: >= 16.0.0
- **npm**: >= 8.0.0  
- **操作系统**: Windows 10+, macOS 10.14+, Ubuntu 18.04+
- **C++交易系统**: 运行在端口 5555 (行情) 和 5556 (交易)

### 1. 安装项目
```bash
# 克隆项目 (如果从git获取)
git clone <repository-url>
cd btrader

# 进入UI目录
cd ui

# 安装依赖
npm install
```

### 2. 演示模式 (推荐首次使用)
```bash
# Linux/macOS
./start-demo.sh

# Windows  
start-demo.bat
```

演示模式会：
- 自动启动模拟数据生成器
- 生成实时市场数据
- 模拟交易响应
- 启动UI界面

### 3. 生产模式
确保您的C++交易系统正在运行，然后：
```bash
npm run dev
```

## ⚙️ 配置说明

### nanomsg 连接配置
编辑 `src/main/nanomsg-bridge.ts`:

```typescript
// 市场数据接收端口
this.marketDataSocket.connect('tcp://127.0.0.1:5555')

// 交易指令发送端口  
this.commandSocket.connect('tcp://127.0.0.1:5556')
```

### 消息协议规范

#### 🔄 行情数据格式 (C++ → UI)
```json
{
  "type": "tick|orderbook|kline|footprint|micro_orderbook",
  "data": {
    "symbol": "BTC/USDT",
    "price": 50000.00,
    "volume": 1.5,
    "timestamp": 1640995200000,
    "side": "buy"
  }
}
```

#### 📤 交易指令格式 (UI → C++)
```json
{
  "type": "place_order|cancel_order|query_position|query_orders",
  "data": {
    "symbol": "BTC/USDT",
    "side": "buy",
    "volume": 1.0,
    "price": 50000.00,
    "orderType": "limit"
  }
}
```

## 🎨 界面功能详解

### 主界面布局
```
┌─────────────────────────────────────────────────────────────────┐
│                    BTrader 订单流交易系统                       │
├─────────────────────────────────────┬───────────────────────────┤
│                                     │                           │
│           K线图表区域               │        交易面板           │
│        (支持多时间周期)             │     (下单/撤单/改单)      │
│                                     │                           │
├─────────────────────────────────────┼───────────────────────────┤
│ 成交明细 │ 买卖档位 │ 足迹图 │ 微盘口 │     持仓状态 │ 委托订单   │
│          │          │        │      │              │            │
└─────────────────────────────────────┴───────────────────────────┘
```

### 核心功能说明

1. **K线图表**: 
   - 实时价格走势显示
   - 支持1分/5分/15分/1小时周期
   - 成交量柱状图
   - 可点击价格快速下单

2. **逐笔成交**: 
   - 最新200笔成交记录
   - 买卖方向颜色区分
   - 点击价格自动填入交易面板

3. **买卖档位**: 
   - 10档深度显示
   - 可视化成交量比例
   - 实时价差显示

4. **足迹图**: 
   - 价格档位成交量分析
   - 买卖力道对比
   - Delta值实时计算

5. **交易面板**:
   - 限价单/市价单支持
   - 快速价格设置 (市价/买一/卖一)
   - 百分比数量设置
   - 一键平仓功能

## 📦 生产环境部署

### 构建可执行文件
```bash

npm run build:win

# macOS
npm run build:mac

# Linux
npm run build:linux
```

### Docker 部署 (可选)
```dockerfile
FROM node:16-alpine

WORKDIR /app
COPY ui/ .
RUN npm install && npm run build

EXPOSE 5555 5556
CMD ["npm", "start"]
```

## 🔧 故障排除

### 常见问题

1. **连接失败**
   - 检查C++系统是否启动
   - 确认端口5555和5556未被占用
   - 检查防火墙设置

2. **数据不更新**
   - 确认nanomsg消息格式正确
   - 检查时间戳格式 (毫秒级Unix时间戳)
   - 验证JSON格式有效性

3. **下单失败**
   - 检查账户余额是否充足
   - 确认交易品种是否支持
   - 验证价格和数量参数

### 调试模式
```bash
# 启动调试模式
npm run dev

# 查看控制台日志
# Electron: F12 开发者工具
# nanomsg: 主进程控制台输出
```

## 🔐 安全注意事项

1. **网络安全**
   - 生产环境请使用加密连接
   - 限制nanomsg端口访问权限
   - 定期更新依赖包

2. **交易安全**
   - 实施订单限额控制
   - 添加交易确认机制
   - 记录所有交易操作日志
